## Приоритет 2 — Качество и стабильность

### 2.1 Миграции базы данных (Alembic)
```bash
cd backend
pip install alembic
alembic init alembic
# Настроить alembic.ini → sqlalchemy.url = sqlite+aiosqlite:///predict.db
```
- **Почему важно:** Сейчас при любом изменении модели нужно удалять БД

### 2.2 PostgreSQL вместо SQLite
- Для production нагрузки SQLite не подходит
- Сменить `DATABASE_URL` на `postgresql+asyncpg://...`
- Добавить в requirements: `asyncpg`

### 2.3 TON webhook вместо polling
- Сейчас: опрос TonCenter каждые 30 сек → задержка до 30 сек
- Альтернатива: webhook через TON API / Tonhub / собственный нода
- **Результат:** мгновенное зачисление депозита

### 2.4 Rate limiting
```python
# pip install slowapi
from slowapi import Limiter
# Лимит: 30 req/min на /trade, 100 req/min на /markets
```

## Приоритет 3 — Новые фичи

### 3.1 Вывод TON (Withdrawal)
- Пользователь указывает сумму → бэкенд отправляет TON со своего кошелька
- Требует интеграции с TON SDK (`@ton/core`) на бэкенде
- Нужен мультисиг или подтверждение для безопасности

### 3.2 История сделок на странице рынка
- `GET /markets/{id}/trades` — последние сделки
- Показывать в Market.tsx под графиком: кто, что, сколько

### 3.3 Категории рынков в Admin
- Выпадающий список вместо текстового поля
- Закрытый набор: Sports / Crypto / Politics / Tech / Other

### 3.4 Автоматический резолв по дате
- При `end_date < now` — статус → `pending_resolution`
- Admin получает уведомление что рынок ожидает разрешения

### 3.5 Поиск рынков
- `GET /markets?q=bitcoin` — поиск по тексту вопроса
- Поле поиска на Home.tsx

### 3.6 Share рынка
- Кнопка "Поделиться" → `tg.openTelegramLink()` с deep link
- Deep link открывает бота на конкретном рынке
